# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Project Dolly is a Streamlit web app that reads multi-regional Google Sheets account planning data, applies strategic scoring logic, generates AI-voiced CRO comments via Claude, and enables a human-in-the-loop review workflow with write-back to Google Sheets.

## Commands

```bash
# Install dependencies
pip install -r requirements.txt

# Run the app
streamlit run app.py

# Regenerate CRO tone calibration from existing Next Steps data
python -m src.ingestion.tone_scraper
```

No test suite is currently configured.

## Architecture

### Data Flow

```
Google Drive → discover regional files by name pattern (config/regions.yaml)
    ↓
Google Sheets API → load Maps, Tech Stake, Summary tabs per region
    ↓
normalize columns (config/column_mappings.yaml) → join into master DataFrame
    ↓
scoring pipeline → NRR + Threading + Expansion → P1/P2/P3 composite tier
    ↓  [cached 30 min via @st.cache_data]
Streamlit UI (pages/) → one-account-at-a-time review card
    ↓
Claude API → AI-voiced comment generation
    ↓
user approve/edit/regenerate → auto-save JSON session + optional Sheets write-back
```

### Scoring Engine (`src/scoring/`)

All thresholds and weights live in `config/scoring_weights.yaml` — tunable without code changes.

- **NRR scorer** (50% weight): CRITICAL (<90% NRR) or renewal <90 days forces P1 override regardless of composite
- **Threading scorer** (25% weight): Single contact = 85 risk pts; dual = 40; 3+ = 0
- **Expansion scorer** (25% weight): Competitor channels (+8 pts each) + whitespace channels (+5 pts each)
- **P-tier cutoffs**: P1 ≥ 65 composite OR hard overrides; P2 = 40–64; P3 < 40

### LLM Integration (`src/llm/`)

- Model: Claude Sonnet 4.6 or Opus 4.6 (user-selectable in UI)
- Prompt system and user templates are YAML-based (`config/prompt_templates.yaml`), not hardcoded
- CRO persona, philosophy, and voice rules injected from `config/cro_persona.yaml`
- Tone calibration (few-shot examples) auto-generated by `tone_scraper.py` from existing Next Steps columns → written to `config/cro_tone_profile.yaml`
- Retry with exponential backoff in `src/llm/client.py`

### Session Management (`src/session/`)

- `SessionState` + `AccountDecision` dataclasses track review order and per-account decisions
- Auto-saved to `data/sessions/session_YYYYMMDD_HHMMSS.json` after each decision
- Sessions are resumable — load a previous JSON to continue where you left off

### Configuration vs. Code

A core design principle: anything a CRO (non-engineer) might want to tune lives in `config/*.yaml`. Scoring weights, NRR thresholds, P-tier cutoffs, prompt templates, CRO persona, column name mappings, and region patterns are all YAML-driven.

### Key Module Map

| Module | Responsibility |
|---|---|
| `src/google/` | Service Account auth, Drive file discovery, Sheets read/write |
| `src/ingestion/` | Load → normalize columns → join regions → scrape tone |
| `src/scoring/` | NRR / threading / expansion sub-scores → composite P-tier |
| `src/llm/` | Anthropic client, prompt builder, comment generator |
| `src/session/` | Session state dataclasses + JSON persistence |
| `src/report/` | Markdown executive report generation |
| `ui/components/` | Streamlit UI widgets (review card, badges, charts, sidebar) |
| `pages/` | Streamlit multi-page tabs (review, dashboard, report) |
